Flujo 1 · Crear un juego

Valida campos requeridos y referencias foráneas antes de guardar el recurso y responder con 201 Created.

@startuml
skinparam backgroundColor transparent
skinparam activity {
  StartColor #22c55e
  EndColor #22c55e
  BarColor #3b82f6
  DiamondBackgroundColor #fbbf24
  DiamondBorderColor #d97706
  DiamondFontColor #111827
  BackgroundColor #3b82f6
  BorderColor #2563eb
  FontColor white
}
skinparam note {
  BackgroundColor #8b5cf6
  BorderColor #7c3aed
  FontColor white
}

start
note: Entrada\nAdmin envía datos del juego\nTitle, Price, DeveloperId, PublisherId, GenreId

if (¿Todos los campos requeridos están presentes?) then (Sí)
  if (¿Existe el DeveloperId en la BD?) then (Sí)
    if (¿Existe el PublisherId en la BD?) then (Sí)
      if (¿Existe el GenreId en la BD?) then (Sí)
        :Guardar juego en BD;
        #10b981:201 Created\nRetornar GameDto;
        stop
      else (No)
        #ef4444:404 Not Found\nGénero no encontrado;
        stop
      endif
    else (No)
      #ef4444:404 Not Found\nDistribuidora no encontrada;
      stop
    endif
  else (No)
    #ef4444:404 Not Found\nDesarrollador no encontrado;
    stop
  endif
else (No)
  #ef4444:400 Bad Request\nDatos inválidos;
  stop
endif
@enduml

Flujo 2 · Obtener lista con filtro

Aplica filtros opcionales por genreId y search, devolviendo 200 OK tanto para resultados como para lista vacía.

@startuml
skinparam backgroundColor transparent
skinparam activity {
  StartColor #22c55e
  EndColor #22c55e
  BarColor #3b82f6
  DiamondBackgroundColor #fbbf24
  DiamondBorderColor #d97706
  DiamondFontColor #111827
  BackgroundColor #3b82f6
  BorderColor #2563eb
  FontColor white
}
skinparam note {
  BackgroundColor #8b5cf6
  BorderColor #7c3aed
  FontColor white
}

start
note: Entrada\nRequest con parámetros opcionales\ngenreId, search

if (¿Se envió parámetro genreId?) then (Sí)
  :Filtrar juegos\nWHERE GenreId = genreId;
else (No)
  :Obtener todos los juegos;
endif

if (¿Se envió parámetro search?) then (Sí)
  :Filtrar juegos\nWHERE Title LIKE %search%;
else (No)
  :Continuar;
endif

if (¿Se encontraron resultados?) then (Sí)
  :Mapear entidades a GameDto;
  #10b981:200 OK\nRetornar List;
  stop
else (No)
  #10b981:200 OK\nRetornar [];
  stop
endif
@enduml

Flujo 3 · Eliminar un juego

Busca el recurso por ID y decide entre 404 Not Found o 204 No Content.

@startuml
skinparam backgroundColor transparent
skinparam activity {
  StartColor #22c55e
  EndColor #22c55e
  BarColor #3b82f6
  DiamondBackgroundColor #fbbf24
  DiamondBorderColor #d97706
  DiamondFontColor #111827
  BackgroundColor #3b82f6
  BorderColor #2563eb
  FontColor white
}
skinparam note {
  BackgroundColor #8b5cf6
  BorderColor #7c3aed
  FontColor white
}

start
note: Entrada\nAdmin envía request con GameId
:Buscar juego por GameId en BD;

if (¿Existe el juego?) then (Sí)
  :Eliminar juego de la BD;
  #10b981:204 No Content;
  stop
else (No)
  #ef4444:404 Not Found;
  stop
endif
@enduml

Compatibilidad

El código funciona bien en renderizadores estándar de PlantUML que soportan diagramas de actividad con if/then/else y coloreado inline.

Ajuste visual

Si quieres un estilo más cercano a BPMN o UML clásico, se puede rehacer con partition, legend y temas personalizados.

Uso académico

También sirve para anexarlo en documentación de endpoints, Swagger complementario o reportes de análisis y diseño de sistemas.
